# Phantom Card Battle v0.1.50

PWA + JavaScript + Canvas / PixiJS で動作する、3×3盤面のカードバトルゲームです。NPC対戦、デッキ編集、図鑑、ショップ、オンライン対戦、ランキング、追加ルール、フィールド効果などを実装しています。

## 1. 現在の主な仕様

### 基本構成

- ゲーム名：Phantom Card Battle
- 動作方式：PWA
- 使用技術：HTML / CSS / JavaScript / PixiJS
- 公開先想定：GitHub Pages
- オンライン機能：Firebase Authentication / Realtime Database
- カード総数：236枚
- 盤面：3×3
- 手札：各5枚
- 数字：1〜10。画面上では10を `A` と表示
- 属性：無属性、もな、美雨、凛花、百花

### タイトル画面

- 対戦
  - NPC対戦
  - オンライン対戦
- デッキ
- ショップ
- 図鑑
- ランキング
- ルール説明
- 設定
- バージョン表示
- 最新版に更新

### NPC対戦

- NPCは進行状況に応じて段階的に解放
- NPCごとに難易度、挑戦料、勝利賞金、使用カード、追加ルール候補、レアチャンス率、初回勝利報酬を設定
- 勝利回数と初回勝利報酬の取得状況を表示
- 難易度によって追加ルールの決まり方が変化
  - よわい：プレイヤーが自由設定
  - ふつう：追加ルール候補から1つ抽選
  - つよい：追加ルール候補①・②から各1つ抽選
- ふつう以上では、挑戦料の半額を支払って追加ルールを再抽選可能
- 挑戦料は追加ルール確定時点で徴収
- 勝利時は挑戦料返金＋勝利賞金
- 引き分け時は挑戦料返金
- 敗北・棄権時は返金なし

### デッキ

- 通常デッキ：5個
- リトル専用デッキ
  - ★デッキ
  - ★★デッキ
  - ★★★デッキ
- 1デッキ5枚
- ★5：1枚まで
- ★4：2枚まで
- ★3以下：制限なし
- デッキ名を設定可能
- 名前・レアリティ・各辺・総合力で並び替え
- ★と属性で絞り込み
- 手持ちカード表示を縦表示／横表示で切替

### 図鑑

- 取得済みカードを表示
- 未取得カードは `???`
- カード名、所持数、★、上下左右の数字、属性枠を表示
- ★と属性で絞り込み
- 番号・名前・レアリティで並び替え
- コンプリート率を236枚基準で計算

### ショップ

- 通貨単位：銭
- 初期所持金：100銭
- ★1〜★3を販売
- ショップ入店時に品揃えを生成
- 品揃え更新は有料
- ショップグレード制度あり
- 累計購入金額に応じて★3の販売比率と更新費用が上昇
- カード売却可能
  - ★4以上は売却不可
  - デッキ使用中のカードは売却不可
  - 買取価格は販売価格の半額
- 販売カードに取得済み／未取得を表示

### オンライン対戦

- 部屋番号制
- ランダムマッチ対応
- 30秒以内に対戦相手が見つからない場合はランダムNPCと対戦
- 報酬、カード移動、銭増減なし
- イロレーティング形式
- 再戦対応
- ランダムで追加ルールが1つ選ばれる
- Firebase Realtime Databaseで盤面・手札・ターン・ルールを同期
- Firebase Anonymous Authを利用
- 対戦中の下スワイプ更新を抑止

### ランキング

Firebase Realtime Databaseに以下を保存・表示します。

- オンライン対戦レート
- 図鑑コンプリート率
- 累計獲得金額
  - NPC勝利賞金
  - カード売却金額
- 同率順位対応
- 対戦成績なしは最下位と同率
- ランキング用ユーザー名を設定可能
- 同一ユーザー名は登録不可
- 使用可能チェックあり

## 2. 基本ルール

1. プレイヤーと相手が交互に3×3盤面へカードを置きます。
2. カードには上・右・下・左の4つの数字があります。
3. 置いたカードの数字が隣接する相手カードの数字より強ければ、相手カードを自分の色に変えます。
4. 9マスすべてが埋まった時点で、自分の色のカード数と残り手札を含めた点数で勝敗を決めます。
5. 通常の数字の強さは `1 < 2 < 3 < 4 < 5 < 6 < 7 < 8 < 9 < A` です。

## 3. 追加ルール

### オーダー

デッキの左から順番にしかカードを出せません。

### カオス

場に出すカードがランダムに指定されます。オーダーとは同時に選ばれません。

### オールオープン

お互いの手札がすべて見える状態で開始します。

### スワップ

開始前にお互いの手札からランダムに1枚ずつ交換します。対戦終了後は元の持ち主へ戻ります。

### リバース

数字の強さが逆になり、Aが最弱、1が最強になります。エースキラーとは同時に選ばれません。ふつう以上のNPCは、リバース時の強さを考慮して手を選びます。

### エースキラー

1だけがAに勝てます。1は2〜9には勝てません。

### タイプアセンド

場に同じ属性のカードが2枚以上あると、その属性の場のカードが強化されます。

- 同属性2枚：各辺+1
- 同属性3枚：各辺+2
- 手札には適用されない
- 無属性には適用されない
- 上限はA

### タイプディセンド

場に同じ属性のカードが2枚以上あると、その属性の場のカードが弱体化されます。

- 同属性2枚：各辺-1
- 同属性3枚：各辺-2
- 手札には適用されない
- 無属性には適用されない
- 下限は1

### プラス

置いたカードと隣接カードの接する辺の合計が、2方向以上で同じ場合に対象カードをひっくり返します。

### セイム

置いたカードと隣接カードの接する数字が、2方向以上で同じ場合に対象カードをひっくり返します。

### コンボ

セイム・プラスなどでひっくり返したカードを起点に、さらに通常比較、セイム、プラスを再判定して連鎖します。

### リトル★／★★／★★★

指定された★以下のカードで対戦します。対応する専用デッキを使用します。他の追加ルールも同時に適用されます。NPCの必須カードは難易度調整のため、指定★を超えていても手札に入る場合があります。

### ミラー

カードを場に出した瞬間に上下・左右が入れ替わります。

- 上 ↔ 下
- 左 ↔ 右
- 手札表示時は通常値
- 場に出した後の比較は反転後の値
- ふつう以上のNPCは反転後の値を考慮

### ワイルドカード

対戦開始時、各プレイヤーの手札からランダムで1枚が選ばれ、次のどちらかの変化が発生します。

- どれか1辺が+2
- どれか1辺がAになり、別の1辺が1になる

変化は手札にある時点から表示され、AIも変更後の数値を使います。

- +2された数値：黄色
- 効果でAになった数値：青色
- 効果で1になった数値：赤色
- +2の結果Aになった場合：黄色

## 4. フィールド効果

### 数字補正

難易度「つよい」のNPC戦では、ランダムで1〜3マスにフィールド効果が発生します。

- -2
- -1
- +1
- +2

そのマスに置いたカードの上下左右すべてに補正が適用されます。補正後の値を1〜Aに丸めてから、通常比較・セイム・プラスを判定します。

### ロック

難易度「ふつう」以上では、0〜1マスがロックマスになります。

- ロックマスはカードを置くまで非表示
- 置いたカードに鍵マークを表示
- ロックされたカードは以後ひっくり返らない

## 5. カードデータ

### CSV

カードデータは `Phantom_Card_Battle_236.csv` で管理します。

主な列：

```text
カード番号,図鑑番号,カード名,レアリティ（星の数）,上,右,下,左,属性
```

### カード番号と図鑑番号

- カード番号：画像ファイルとカードIDの紐づけに使用
- 図鑑番号：ゲーム内のNo.表示に使用

例：

```text
カード番号 183 → assets/cards/card_183.webp
図鑑番号 120 → ゲーム内では No.120 と表示
```

### JavaScriptデータ

CSVから `src/data/cards.js` を生成します。

```js
{
  id: "card_001",
  cardNo: "1",
  no: "1",
  encyclopediaNo: "1",
  name: "おともだち",
  rarity: 1,
  up: 5,
  right: 5,
  down: 2,
  left: 2,
  power: 14
}
```

## 6. NPCデータ

NPC設定は `src/data/npcs.js` で管理します。

v0.1.45から、以下はカードIDや図鑑番号ではなく、カード名を基準に解決します。

- 初回勝利報酬
- 必須カード
- 固定カード

同名カードがある場合は、属性・レアリティ・カード番号を併用して特定できます。

```js
firstWinRewardCardRef: {
  name: "[キミ×]",
  type: "もなタイプ",
  rarity: 4
}
```

起動時に `resolveCardRef()` が現在のカードリストから該当カードを検索し、実際のカードID・図鑑番号へ変換します。

### 属性・レアリティによるカードプール

```js
cardPoolSpec: {
  rarities: [2, 3],
  type: "もなタイプ"
}
```

カード追加や図鑑番号変更後も、指定条件に一致するカードを自動で候補に含めます。

## 7. カード画像

カード画像は `assets/cards/` に保存します。

```text
assets/cards/card_001.webp
assets/cards/card_002.webp
...
assets/cards/card_236.webp
```

画像にはイラストだけを用意し、カード名・★・上下左右の数字はアプリ側で重ねて表示します。オリジナル画像へ差し替える場合は、同じファイル名で上書きしてください。

推奨仕様：

- 形式：WebPまたはPNG
- サイズ：512×512px
- 比率：正方形
- 重要な人物や物体は中央寄り

## 10. Firebase設定

### 必要なサービス

- Firebase Webアプリ
- Authentication
  - Anonymousを有効化
- Realtime Database

PCで常時サーバーを起動する必要はありません。GitHub PagesからFirebaseへ直接接続します。

### firebase-config.js

FirebaseコンソールのWebアプリ設定を `firebase-config.js` に入力します。

```js
export const firebaseConfig = {
  apiKey: "...",
  authDomain: "...firebaseapp.com",
  databaseURL: "https://...firebasedatabase.app",
  projectId: "...",
  storageBucket: "...firebasestorage.app",
  messagingSenderId: "...",
  appId: "..."
};
```

### Realtime Database Rules例

以下は現在のカジュアル運用向けの簡易例です。

```json
{
  "rules": {
    ".read": false,
    ".write": false,
    "rooms": {
      "$roomId": {
        ".read": "auth != null",
        ".write": "auth != null"
      }
    },
    "matchmaking": {
      ".read": "auth != null",
      ".write": "auth != null"
    },
    "profiles": {
      "$uid": {
        ".read": "auth != null",
        ".write": "auth != null"
      }
    },
    "usernames": {
      "$nameKey": {
        ".read": "auth != null",
        ".write": "auth != null"
      }
    },
    "leaderboards": {
      "$kind": {
        ".read": "auth != null",
        "$uid": {
          ".write": "auth != null"
        }
      }
    }
  }
}
```

このルールは不正操作を完全には防ぎません。報酬付きオンライン対戦や厳密なランキングを実装する場合は、Cloud Functions等でサーバー側判定を行ってください。

## 14. 注意事項

- ブラウザのサイトデータ削除はセーブデータ消失につながります。
- Service Worker更新後に表示が古い場合は、アプリ内の「最新版に更新」を使用してください。
- Firebase設定値はWebアプリから参照できる公開設定ですが、Realtime Database Rulesは必ず設定してください。
- 現在のオンライン対戦・ランキングはブラウザ側処理を含むため、本格的な競技運用や報酬付き運用には追加の不正対策が必要です。
- 公開時にはカード画像、名称、BGM、効果音等の権利関係を確認してください。
