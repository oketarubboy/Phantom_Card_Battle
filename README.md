# Phantom Card Battle v0.1.19

PWA + JavaScript + Canvas / PixiJS で作成した、3×3カードバトルの初期版です。

## 内容

- タイトル画面
- 対戦画面
  - 3×3盤面
  - 自分の手札5枚
  - 相手の手札5枚は裏向き表示（よわいNPCは公開表示）
  - 現在ターン、スコア、ログ表示
  - カード配置・反転エフェクト
  - カード画像表示対応
  - スマホ表示時のAutoFit対応
- デッキ画面
  - デッキ1〜5保存
  - 手持ちカードの名前順・レアリティ順・上下左右/総合強さ順の昇順/降順並び替え
  - ★5は1枚まで
  - ★4は2枚まで
  - ★3以下は無制限
  - カード画像表示対応
- 図鑑
  - 取得済みカード表示
  - 未取得カードは「???」表示
  - カード画像表示対応
- 設定
  - エフェクトON/OFF
  - セーブデータのエクスポート/インポート
- PWA
  - manifest
  - service worker
  - バージョン表示
  - 最新版に更新ボタン

## 起動方法

ローカルで確認する場合は、フォルダ内で下記のように簡易サーバーを起動してください。

```bash
python -m http.server 8000
```

その後、ブラウザで開きます。

```text
http://localhost:8000/
```

## カード画像について

182枚分の仮サンプル画像を `assets/cards/` に同梱しています。

画像ファイル名はカードIDと一致しています。

```text
assets/cards/card_001.webp
assets/cards/card_002.webp
assets/cards/card_003.webp
```

今後オリジナル画像に差し替える場合は、同じファイル名で上書きしてください。

例：`card_001.webp` を差し替えると、カードID `card_001` の画像が変更されます。

## カードデータについて

`Phantom_Card_Battle_182.csv` から `src/data/cards.js` を生成しています。

カード番号は `card_001` 〜 `card_182` の形式でアプリ内IDに変換しています。

## 公開時の注意

カード名に既存作品由来と思われる名称が含まれている場合、公開前にカード名・絵柄・BGM・効果音などはオリジナル化してください。

## Firebase設定
`firebase-config.js` にFirebase Webアプリの設定値を入力してください。Firebaseを使うため、どこかのPCでサーバーを起動する必要はありません。

Realtime Database Rulesの開発用例：
```json
{
  "rules": {
    "rooms": {
      "$roomId": {
        ".read": "auth != null",
        ".write": "auth != null"
      }
    }
  }
}
```
公開運用では、必要に応じてルールを厳しくしてください。

### Realtime Database Rules追加例
オンライン対戦のレート更新をブラウザ側で行うため、まずは下記のルールで動作確認してください。

```json
{
  "rules": {
    ".read": false,
    ".write": false,
    "rooms": {
      "$roomId": {
        ".read": "auth != null",
        ".write": "auth != null"
      }
    },
    "profiles": {
      "$uid": {
        ".read": "auth != null",
        ".write": "auth != null"
      }
    },
    "usernames": {
      "$nameKey": {
        ".read": "auth != null",
        ".write": "auth != null"
      }
    },
    "leaderboards": {
      "$kind": {
        ".read": "auth != null",
        "$uid": {
          ".write": "auth != null"
        }
      }
    }
  }
}
```

※このルールはカジュアル対戦向けです。報酬あり・不正対策ありにする場合は、Cloud Functions等でサーバー側判定に変更してください。
